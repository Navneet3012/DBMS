PL/SQL Data Types

Each value in PL/SQL such as a constant, variable, or parameter has a data type, which defines a storage format, constraints, and a valid range of values.

PL/SQL Categories of Data Types

  • Scalar types (e.g., number, character, Boolean, date)
  • Composite types (e.g., records, tables)
  • Reference types (e.g., pointers)
  • Large Object (LOB) types (e.g., images, videos, audio)

PL/SQL Scalar Data Types

Scalar data types hold a single value and have no internal components. These include:

Data TypeDescription
NUMBER(p,s)Stores fixed or floating-point numbers with optional precision (p) and scale (s)
CHAR(n)Fixed-length character data
VARCHAR2(n)Variable-length character data
BOOLEANRepresents logical values TRUE, FALSE, or NULL
DATEStores date and time values

Syntax

variable_name datatype [NOT NULL] [:= initial_value];

Example

DECLARE age NUMBER(3); name VARCHAR2(50); is_valid BOOLEAN := TRUE; BEGIN NULL; END;

PL/SQL Composite Data Types

Composite types have internal components that can be manipulated individually. Examples include:

  • RECORD: Group of related items as a unit
  • TABLE: Collection of data like a single-dimensional array
  • VARRAY: Variable-size array

PL/SQL Reference Data Types

Reference types hold pointers or references to other program items. These include:

  • REF CURSOR: Pointer to a query result set

PL/SQL Large Object (LOB) Data Types

LOB types are used to store large volumes of unstructured data like text, images, video, etc.

  • BLOB: Binary Large Object
  • CLOB: Character Large Object
  • NCLOB: NCHAR Character Large Object
  • BFILE: Binary file stored in the file system

Summary

PL/SQL provides a rich set of data types that help manage structured and unstructured data efficiently. Choosing the right data type is essential for memory optimization, performance, and data integrity.
